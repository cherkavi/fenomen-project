реализация транспорта с COM портом посредством прямого доступа в порт, то есть создается объект SerialPort и с ним начинается информационный обмен. ВАЖНО: нельзя использовать в системах 24/7, основная причина - при вынимании устройства из порта происходит "обвал/зависание" программы, необходимо использование другой реализации (через запуск Reader-а SerialPort-a как внешней программы и контроля ее с помощью TCP портов )
